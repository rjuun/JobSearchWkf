#7Interview brief called covered requirements "thin spots"suggestion → truthfulness

✓ fixed

Bridges came purely from a requirement's initial match strength, ignoring whether the user later kept green evidence for it in C2. #5 · Assembled matrix vs NEXT_COVERAGE_MATRIX ✓ coupled

With NEXT_GRAPH_ASSEMBLED=1 and NEXT_COVERAGE_MATRIX=0, the coverage matrix still showed as the assembled /profile face while its dedicated page and nav button were gone.

Decision — couple them. The assembled view is a coverage-matrix presentation; retiring the matrix must not resurrect it as the profile's primary face. The dependency is real and directional (assembled ⊃ matrix).

Shipped: canAssemble = nextGraphAssembled && nextCoverageMatrix now gates both the assembled face and its view-toggle; with the matrix off, /profile degrades cleanly to the strength-meter view. app/profile/page.tsx

#9 · Excavation "Not now" — snooze or dismiss? ✓ real snooze

"Not now" called skipPromptAction — a permanent one-way skip — while the telemetry read excavation · snoozed. A dismiss wearing a "later" label.

Decision — make it a genuine snooze. Rediscovery's whole value is that a thin era keeps gently resurfacing until you mine it; losing that invitation forever on one busy-day "Not now" defeats the feature.

Shipped: "Not now" now hides the card for the session only — it returns next visit. The prompt stays open (generatePrompts dedupes on its key, so nothing duplicates), and the snoozed event name is now truthful. Lightest honest version — no migration. components/coach-queue.tsx
