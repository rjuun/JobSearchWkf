## The value scale

| Source | Value | Rationale |
|---|---|---|
| `screening_gap` | 95 | A live watch-out on a real role — most valuable to resolve |
| `target_requirements` | 70 + (Core 12 / Important 4) + min(roles, 4)×2 → **76–90** | Demand-pull, weighted by rank and how many roles ask; capped below screening |
| spawned follow-up | 60 | A timely deeper question after an approval |
| prior-role metric | 55 | A real story missing a number |
| prior-role summary | 45 | A role missing its one-line scope |
| similar-resume pattern | 35 | A competence peers commonly show |
| ATS bundle | 25 | Housekeeping — real, but lowest priority |

## Code review (agent, simplify pass) — findings & resolutions

The reviewer confirmed the value math (screening always ≥ target, ties stable, no float drift), the
migration/`toRow`/`rowChanged` wiring, and no queue-consumer regressions. Fixes applied:

| Severity | Finding | Resolution |
|---|---|---|
| Med-High | Constant `e1:ats-bundle` key made the bundle a one-shot — skills you skipped or added later never re-prompted (a `done` row with that key is never re-inserted) | Key now derived from the covered skill set via a stable hash — a changed set yields a fresh prompt; the old one auto-closes |
| Medium | Bundle regex split on the first separator, truncating hyphenated names ("Cross-functional leadership" → "Cross") and silently dropping the line | Parse by matching the **known** skill names (longest first), then strip the separator |
| Low | Em-dash `—` wasn't an accepted separator | Added to the separator set |
| Low (noted) | `committedNodeId` records only the first merged skill | Acceptable for now — noted |

The dead single-`ats` commit branch is kept for backward-compat with pre-M2 rows.
