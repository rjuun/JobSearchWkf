## Code review (agent, simplify pass) — findings & resolutions

The reviewer verified the client state machine (auto-advance, Enter handler, toast timer, hooks) is clean,
and `delete-star`/`delete-results` fully remove their rows. Fixes applied:

| Severity | Finding | Resolution |
|---|---|---|
| Medium | Undoing an approval that had **spawned a follow-up** deleted the follow-up's prompt row but left its committed evidence + answer rows orphaned | `undoApprovalAction` now fully purges spawned children (revert their evidence + delete their answers) before deleting them |
| Medium | Revert restored the prior *value* of a summary/ATS row but not its prior **`source`** — a `manual`/`imported` row stayed stamped `ai_coached` | The revert plan now captures & restores `priorSource` |
| Low-Med | `approveAnswerAction` wasn't idempotent (a double-submit could double-commit) | Added a `status === 'done'` guard |
| Low | Undo left a phantom strength snapshot and didn't regenerate the queue | Undo now re-snapshots + regenerates (mirrors approve's tail) |
| Low | `approveAnswerAction` returned a value the client ignored | Dropped the return |

Reviewer's cosmetic notes (one-render undo flicker) left as-is.

## Testing steps (manual)

1. `npm run seed && npm run dev` → `/profile/coach`.
2. The stage opens on the highest-value prompt ("Highest value now"). Answer → **Draft** → **Enter** (or Approve).
3. The next prompt loads itself; the rail tally shows "1 captured · +N" and the meter climbs.
4. Click **Undo** in the toast (or the tally) → the capture reverts, the prompt returns.
5. After 3 captures → a stop card ("Nice set — 3 captured"); **Keep going** continues.
