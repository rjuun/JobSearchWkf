## The metric

- **time-to-first-CV** = first `cv_generated` − first `paste`/`warmup`.
- **decisions-before-win** = `keep` events between the first start and the first CV (inclusive window).
- Both are `null` until a CV is generated; the dashboard copy states which stage the user is at.

## Remaining gaps / follow-ups

- Metrics are **per-owner** (single-tenant demo). A real cohort view (percentiles across users, funnel
  drop-off) is a deploy-time analytics concern, not built here.
- `cv_generated` fires on every generate; time-to-first-CV uses the *first* one, so re-generations don't
  skew it, but `totalKeeps`/`evidence kept` counts all green approvals (intended).
- The warm-up path commits the CV as imported history (same as M4) and routes to the coach; the "2–3
  warm-up questions then guided paste" is delivered via the existing coach rail + a nudge rather than a
  dedicated stepper.
