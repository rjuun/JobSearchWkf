## The invariant

`provenanceCoverage(rows)` → `{ green, traced, complete }`: `complete` iff every Kept (green) line carries an
`evidenceRef`. The CV guarantee and the "100% traced" ledger are rendered from this — computed, never asserted.
Verified on the seeded ready CV: **12 green / 12 traced / complete**.

## Code review (agent, simplify pass) — findings & resolutions

Reviewer confirmed the migration, `getTailoring` column flow, `provenanceCoverage` edge cases, graceful
degradation of seeded rows, and the docx swap are all clean. Fixes applied:

| Severity | Finding | Resolution |
|---|---|---|
| Medium | The CV generates from **all** green rows, but the proof trail listed only rows with a ref and its summary said "100% traced" — so an untraced Kept line was silently omitted while the guarantee showed a caution (self-contradiction) | The trail now lists **every** Kept line; untraced ones show a "no source / source pending" marker; the summary + first bullet are gated on `cov.complete` (show "{traced} of {green}" when incomplete) |
| Low | `approvedAt` re-stamped on every green write, losing the original date | Stamp only on the transition into green (mirrors the keep-event guard) |
| Low | `LeftOutCard` could render an empty body for a red row with only a requirement line | Filter now requires an actual body (`originalText`/`cvBullet`) |
| Nit | `tracedCount` duplicated `cov.traced` | Removed; the card reads from `cov` |

## Remaining gaps / follow-ups

- **`prov_source` is set at the row's origin** (default `imported`); a fuller model would set `coached` when a
  line came from the coach and `swapped` on an evidence swap. The column + UI are in place; richer tagging is
  a follow-up.
- **"Proof view once, then Reader" persistence** is delivered via the collapsible "Show sources" toggle rather
  than a per-lead persisted mode — the guarantees are always visible from first paint; a saved reader/proof
  preference is a small follow-up.
- A **live visual pass** of the proof trail + Left-out card is recommended (as with M3/M6).
