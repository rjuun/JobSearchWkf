The cold start now answers the user's question, not ours · ✓ Complete

✨ The one-line

New users used to hit a curation wall — draft 24–60 items and Keep/Drop every one before seeing anything. Now they paste a CV + one job ad and get a real Role-Fit verdict in a couple of minutes; the Career Graph is born from it, and they curate only the ~6 lines the CV actually needs.

🔀 Before → after

paste → draft 24–60 → curate ALL → then find a role
paste CV + job ad→ verdict (from your history)→ Keep ~6 for the CV→ download CV

≈ 8 decisions, no forms — inside the ≤10 / ≤5-min activation bar.

✅ What I changed

✓
A real front door at /start — two paste boxes (with samples) → "Screen this role". Established users skip it.
✓
One action wires it all: extract graph from the CV → commit as imported history → create the lead → screen → queue the coach's first session → land on the verdict.
✓
Signup routes here, not the old wall. The wall stays only as an "import more" path.
✓
Honest label: "your Career Graph is born — this verdict is based on your imported history; it firms up as you approve."

💭 A scope call to sanity-check

?
I commit the imported CV as history up-front and screen against it (labelled), rather than the purest "every node stays draft until you Keep it" model — which would need a draft/approved flag on every evidence row. This reuses the whole existing pipeline and hits the activation bar; the truer per-Keep-of-draft version is noted as a follow-up. Flag if you want the stricter model.
