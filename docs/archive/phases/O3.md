# O3 · Coach & enrich (drawing out the under-told)

> Detailed record of phase **O3** from the [user-journey roadmap](../design/USER_JOURNEY.md).
> Friendly review (with screenshots): [`o3-coaching.html`](o3-coaching.html).

O1 made the Career Graph visible; O2 made it fast to build. **O3 makes it stronger** — it spots
what's under-told and asks one question at a time. The highest-value coaching for senior profiles
is drawing out the *impact they omit*; O3 does exactly that, and tags where every node came from.

## The coaching loop

```
graph ─▶ enrichmentTargets()  ─▶  one question at a time  ─▶  answer becomes evidence
          (code computes the gaps)    (metric / ATS / summary)    (tagged ai_coached)
```

The gaps are **computed in code** — no LLM needed, so coaching is deterministic, works offline, and
can't hallucinate a question. Three target kinds:

| Kind | Found when | The question | The answer becomes |
| --- | --- | --- | --- |
| **metric** | a story has no quantified result | *"What was the measurable impact of …?"* | a new `star_result` (with the number, if you give one) |
| **ats** | a skill has no ATS keyword variants | *"How else might a job ad phrase …?"* | merged `ats_keyword_variants` on the skill |
| **summary** | a position has no summary | *"In one line, what did you own as …?"* | the position's `summary` |

**Truthfulness holds:** a coached result only gets a `metric` when the answer actually contains a
number (`extractMetric`) — the coach never invents one.

## Provenance

Every evidence row now carries **`source`** (`authored` · `imported` · `ai_coached`) and a
`confidence`. Imported nodes (O2) are tagged `imported`; coaching tags what it touches `ai_coached`;
hand-entered evidence stays `authored` (no badge). The section editors show a small badge so you can
always see how a piece of evidence entered your graph.

## Key decisions

- **Code computes the gaps.** Matching the system's rule (the LLM judges, code computes), the
  enrichment targets are deterministic. An LLM could *phrase* warmer questions later; the gap
  detection shouldn't depend on it.
- **Coaching produces honest evidence.** The answer is the user's own words; the coach only asks.
  A metric is captured only if stated.
- **Provenance is visible, not hidden.** "imported" / "AI-coached" badges build trust — you can
  audit every claim's origin.

## Verification

- `tsc --noEmit` and `next build` green (`/profile/coach` builds).
- **Live round-trip** (`scripts/verify-o3.mjs`): import the sample → strength drops to **87/100**
  with **8 opportunities** → answer one → the skill gains ATS variants and turns `ai_coached` →
  graph restored to baseline (all `authored`) afterwards.

## Gaps · missing · future

- **LLM-phrased coaching.** The questions are templated. A `suggestEnrichments` LLM pass would rank
  and phrase them more naturally (and spot subtler gaps) — wired the same `runStructured` way when a
  key is available.
- **Gap → enrich from screening.** Mission control flags "Core requirement · Weak match" during
  tailoring; those should deep-link straight into a coaching prompt. Not wired yet.
- **Confidence isn't used yet** to prioritise — low-confidence imported nodes are good coaching
  candidates ("is this right?").
- **Only three target kinds.** Competences, attributes and thin stories (few actions) could also be
  coached.
- The `extractMetric` parser is deliberately simple (first number-ish token).
