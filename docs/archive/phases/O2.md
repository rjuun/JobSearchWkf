# O2 · Import → extract → curate (the bootstrap half of onboarding)

> Detailed record of phase **O2** from the [user-journey roadmap](../design/USER_JOURNEY.md).
> Friendly review (with screenshots): [`o2-onboarding.html`](o2-onboarding.html).

O1 made the Career Graph **visible and editable**. O2 makes it **fast to build**: paste a CV or
LinkedIn export, the AI drafts a graph, and you approve what's true. It applies the system's own
best pattern — **AI drafts, the human curates** — to profile-building, so a new user reaches a
usable graph in minutes instead of filling a 12-sheet workbook by hand.

## The flow

```
Paste / upload ─▶ extract (LLM or mock) ─▶ stage (draft_graph) ─▶ curation gate ─▶ commit
                  emit_career_graph         onboarding_state       Keep / Drop      → real tables
```

Nothing the AI produces touches the trusted graph until you approve it. The draft lives in a
**staging** column; only the nodes you keep are promoted, with generated `ref_code`s, into the real
evidence tables.

## Key decisions

- **Mock-first.** No API key is needed: the heuristic extractor is deterministic, so the whole flow
  demos and verifies offline. The live path is wired behind `LLM_MODE=live`.
- **Truthfulness guardrail, in code and prompt.** A result only gets a `metric` when the text
  actually contains a number — the extractor never invents one. (You can see the metric badges in
  the gate light up only for "6 countries / 22% / 8% / 4.5m".)
- **Staging, not a graveyard.** Unapproved nodes stay in `onboarding_state.draft_graph`; the trusted
  tables only ever receive approved evidence — the same `pending → green` discipline as the C2
  tailoring gate.
- **Curation = Keep / Drop** (with confidence shown). Editing a node before keeping is deferred —
  you can edit freely in the O1 section editors after commit.
- **Never clobber identity.** If you already have a profile, an imported identity is shown but not
  written over.

## Gaps · missing · future

- **File parsing.** Today the import is **paste-text**; PDF / DOCX / native LinkedIn-export parsing
  is not wired (a `pdf-parse`/`mammoth` step is the obvious next addition).
- **Live extraction is unexercised** (no API key here). The mock heuristic is intentionally simple —
  it groups all bullets under one "Imported highlights" story and won't always split skills cleanly.
  The live LLM would produce real STAR groupings and richer skills; that needs a key + prompt tuning.
- **Edit-during-review** isn't in the gate yet (Keep/Drop only) — you edit post-commit in O1.
- **No dedup against the existing graph** — importing can create near-duplicates of evidence you
  already have. A "looks like X you already have" check is a future safeguard.
- **Provenance** (`source='imported'`, `confidence`) is captured in the draft but **not yet stored**
  on committed rows — that lands in **O3**, where it's surfaced as badges and feeds coaching.
