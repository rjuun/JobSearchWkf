Isolation, proven

How — one primitive: currentOwnerId() returns the logged-in user's id, and every query is scoped to it while every write stamps it. getLead(id) is scoped too, so you can't load someone else's lead by guessing its id. This is literally owner_id = auth.uid() — so on Supabase, drizzle/rls.sql turns it into database-enforced, defence-in-depth isolation.

How it maps to Supabase

Built and verified on local Postgres — real users, real sign-up, real isolation. The deployment swap is auth-provider + RLS, not a re-architecture: the local users table becomes Supabase's managed auth.users, auth.uid() returns our owner_id, and one generated policy (drizzle/rls.sql) enables owner_id = auth.uid() on every owner-scoped table. Password hashing uses Node's crypto (scrypt) — no new dependency.
