JobSearch Camunda · Experience & Design System

Quiet confidence,
a journey you can feel

The prototype worked — but it looked like a generic admin and read like a flat data dump. This phase gave it an intentional design language and turned the staged, gated pipeline into the felt experience: a visible spine, a score-forward cockpit, one obvious next move.

Inter + JetBrains Mono Semantic tokens Variant component kit Journey rail A11y + reduced-motion

The two decisions

Aesthetic

Quiet confidence

Calm, premium, restrained — Linear / Stripe-dashboard energy. The system is the product, so the chrome steps back. Colour is rationed; spacing, depth and type do the work.

Journey

Re-architected

The A→B→C→D pipeline became a visible spine: a persistent stage rail, a score-forward "mission control," and a single, obvious next action at every step.

Before → After

● Before

  • Tailwind defaults + one indigo; no real token system
  • Status colours duplicated in two files
  • No Button/Input components — every control bespoke
  • Lead detail = a flat stack of equal-weight cards
  • Fit score a tiny pill in the corner; no sense of progress
  • No focus rings, sparse ARIA

● After

  • Semantic surface/ink tokens + refined brand scale
  • One registry (lib/ui.ts) feeds every badge & bar
  • Variant kit: buttons, fields, badges, score, rail…
  • Mission control: hero → rail → next action → current step
  • Score leads the hero; the rail shows where you are & what's gated
  • One quiet focus ring everywhere; ARIA on the gate votes

lib/journey.ts collapses a lead's status + score into this four-stage arc — marking each stage done / current / locked / upcoming, computing the one next action, and locking Tailor when a lead screens below the bar. The UI only renders it.
