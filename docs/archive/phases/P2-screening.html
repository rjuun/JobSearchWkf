★The principle that makes it trustworthy

The LLM judges. The code computes.

Each step forces a single tool call with a strict schema (validated by zod, one retry on failure). B6 returns only judgments — 0–10 ratings per dimension and per requirement. Every rollup and gate is plain TypeScript, so the same inputs always yield the same score.

overall = 0.35·relevance + 0.20·seniority + 0.20·impact + 0.15·reqAlignment + 0.10·ats reqAlignment = Σ(reqScore·weight) / Σ(weight) // Core 3 · Important 2 · Nice 1 recommend = ≥7 Proceed · ≥5.5 Caution · ≥4 Low priority · else Not recommended

2What was built

Anthropic client

One choke point, two modes

Forces a tool call, validates with zod (+retry), logs tokens. Mock mode runs the whole pipeline with no API key; live mode calls Claude — same contract either way.

Scoring

Deterministic rollup

All weighted-average math, requirement alignment, recommendation tiers and the B1 hold gate live in lib/scoring.ts — never the model.

Prompts

Your notes are the prompts

Each step loads its Process/*.md note as the system prompt — refining a step means editing markdown, not code.

Capture

Bookmarklet + paste

A public ingest endpoint for the LinkedIn bookmarklet, plus a manual paste form — both create a lead and store the JD.

Console

Run it live

"Run screening" shows each step completing with its model + latency, then the score and the promote-to-tailoring gate.

Audit

Every run recorded

pipeline_runs + llm_calls capture each step's output, model and token use — the basis for the cost panel in P4.
