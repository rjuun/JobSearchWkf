1What was built

Database

Drizzle schema + migration

24 tables & 4 enums covering the evidence side (STARs, bullet bank, skills) and the pipeline side (leads, requirements, tailoring), generated & applied to Postgres.

Auth

Single-user session gate

jose-signed cookie + edge-safe lib/session.ts; middleware gates every route except /login and the bookmarklet endpoint.

Seed

Workbook → Postgres importer

One idempotent script reads both Excel workbooks + 25 JD markdown files, linking requirements and captured JDs to their leads.

Storage

Filesystem adapter

Bucket-relative read/write API for JD captures & CVs — maps 1:1 to Supabase Storage in production.

UI

Lead board & detail

Sortable board with fit scores + a rich lead detail (screening snapshot, 17-skill grid, requirements table, captured JD).

Client

Hot-reload-safe DB

Drizzle over postgres.js as a singleton, with typed query helpers and the B6 rank-weight map ready for scoring.

5Key decisions

Local Postgres, Supabase-shaped

No Docker on the build box → the demo runs on Homebrew Postgres via Drizzle. Supabase is Postgres, so the same schema + DATABASE_URL point at the cloud in P5 with no code change.

Adapters for auth & storage

Thin lib/auth.ts and lib/storage.ts mean swapping to Supabase Auth/Storage later is an implementation detail, not a rewrite.

owner_id everywhere, RLS later

Every table carries owner_id (one demo owner). App-layer scoping now; Supabase RLS owner_id = auth.uid() is the clean multi-user step.

Index-based seed parsing

Banner rows + multi-word headers make the workbooks fiddly; the importer reads by fixed column index and maps free-text status to the enum heuristically.

6Polish — the simplify pass

Extracted lib/session.ts so the session cookie name + signing key live in one place (they were duplicated across middleware/auth/env — a real drift hazard). The seed now captures JDs before inserting leads and links on insert, removing ~20 sequential UPDATE round-trips; lead-detail I/O was parallelized; and a handful of null-checks and helpers were tightened. Intentionally kept: the cn() className seam and the UI status-style map.
