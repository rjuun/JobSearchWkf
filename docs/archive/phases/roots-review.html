Executive verdict

You set out to turn a staged, gated, evidence-bound job-search methodology (Obsidian step notes + Excel/SharePoint + Claude by hand) into a runnable app. The prototype honours the spine: capture → screen with a reproducible Role Fit Score → human-gated tailoring → downloadable CV. It also added something the manual process never had in-product: a Career Graph you can build, coach, and grow (O1–O4).

The biggest drift from your roots is not the UI rebrand — it's depth of automation fidelity: tailoring (C) is still largely mock/code rather than the full Process/C*.md prompts; CV output uses a programmatic layout instead of your Word template; and the CI dashboard was folded away when /dashboard was removed. ↳ Resolved. The tailoring core (C2/C3/C5/C7) now runs live on DeepSeek over the whole evidence graph; C6 fills the real CV_Template.docx (now incl. the tailored profile); the CI dashboard is restored at /dashboard; B3 reads your Values & Motives; the ATS score persists across reloads; and live runs show progress feedback. See what changed.

Bottom line — The architecture was right and held: LLM judges, code computes, Process notes are prompts, humans gate. The prototype now runs the methodology end-to-end on live LLM with template-fidelity output — it is demo-ready. Remaining work is depth (role-dynamic skills, CV PDF preview, D-phase tracking, live calibration), not a rebuild.

What's confusing

Naming
JobSearch_Camunda → RoleProof

Repo and docs still say JobSearch_Camunda; the app brand is RoleProof. README demo path still references /leads and /dashboard which no longer exist.

Vocabulary
Keep / Maybe / Drop vs Green / Yellow / Red

DB and code use green|yellow|red; UI says Keep/Maybe/Drop. Fine for users, but breaks 1:1 mental mapping when comparing to Process notes.

Hidden steps
B1–B6 / C1–C7 invisible in the workspace

By design (plain-language UX), but you lose traceability to the methodology you authored. No "expert mode" toggle to show step IDs, ref codes, or model used.

Two layouts
Command center (2a) vs spine (2c)

Same pipeline, two interaction models. Unclear which is canonical; doubles maintenance and review surface.

Two homes
Career Graph vs Leads board

Correct architecturally (evidence vs applications), but new users may not know which to open first. Onboarding ends at graph; lead flow starts at board — the link is implicit.

Mock vs live
Everything works without an API key

Great for demos; easy to mistake mock output for production-quality tailoring. No in-app indicator of LLM mode.

CI split
Improve tips vs CI initiatives

Tips are visible; the 10 seeded ci_initiatives (the Obsidian dashboard equivalent) are not surfaced anywhere.
