JobSearch Camunda · Retrospective

What we built, decided & what's next

Five phases turned a document-driven job-search methodology into a runnable, demo-ready SaaS prototype — screening leads into a reproducible fit score and tailoring evidence-backed CVs with a human in the loop. Here's the full picture: decisions, gaps, and the road ahead.

2Key decisions

DecisionWhy
LLM judges, code computesThe Role Fit Score must be reproducible — the model emits 0–10 judgments; all arithmetic is TypeScript.
Mock-first LLMRun & verify the whole pipeline offline with no API key; deterministic demos. The live path switches on by env.
Your Process notes are the promptsRefining a step = editing markdown, not code — keeps the improvement loop meaningful.
Human-gated tailoringOnly Keep-approved, evidenced claims reach the CV — the system's trust differentiator.
Local Postgres + DrizzleNo Docker needed; Supabase is Postgres, so the same schema/URL go to the cloud unchanged.
Single-user custom authFastest path to a gated demo; schema carries owner_id so multi-user (Supabase Auth + RLS) is a clean step.

3Gaps & missing elements

Out of scope (prototype boundary)

  • Multi-tenant signup (single-user by design)
  • D-phase: application + target-company monitoring
  • Automated, values-aligned lead discovery
  • Cover-letter generation & B7 summary
  • Camunda 8 runtime (BPMN stays documentation)

Simplifications to deepen

  • CV: programmatic docx → fill the real template via docxtemplater
  • Group achievements by role (not one section)
  • C7 should be an Opus judgment, not a coverage metric
  • C2 live path should use the Sonnet prompt (not token overlap)

Data-model gaps

  • Add skill_category (C4 groups by proficiency for now)
  • Persist the C7 rating + ATS breakdown on the lead
  • Log applications records

Operational

  • Live LLM unexercised — tune prompts against the real API
  • RLS policies not written (app-layer scoping for now)
  • No unit/e2e tests beyond the 3 verifiers
  • Bookmarklet host hardcoded; dashboard aggregates in JS

5Run it yourself

createdb jobsearch_camunda
npm install
npm run db:migrate && npm run seed     # schema + your real data
npm run dev                            # http://localhost:3000

npx tsx scripts/verify-screening.ts    # B1–B6 + deterministic rollup
npx tsx scripts/verify-tailoring.ts    # C1–C7 + a valid 2-page .docx

Demo path: /leads → open a lead → Run screening → Promote → Map evidence → approve (Keep) → Generate CV → Download. The /dashboard tells the whole story.
