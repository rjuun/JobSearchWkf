#### Practical Rules I Suggest

- **First pass**: Keep all bullets linked to **Core** requirements.
- **Second pass**: Keep the strongest bullets linked to **Important** requirements (based on impact and seniority).
- **Third pass**: Review Nice-to-Have linked bullets — only keep if they add unique value and space allows.
- When in doubt between two similar bullets → Keep the one with **stronger impact / seniority signal**.
- For older roles (e.g. Tokyo), be more aggressive with reduction (as you already noted).

I kind of would like to propose a different way to practically do this prioritisation.

My way of doing this would be to create a table in which we have

-          Job Requirements ID
-          Requirements Order
-          Rank
-          Requirement Group
-          Requirement
-          Requirement Description
-          Requirement Skills
-          My Skills
-          CV Position (ranked from A0 to E1
-          CV Bullets, with the 23 CV bullets we have worked on.
-          one column for each of the criteria that you have correctly pointed out
	o   Core Requirement alignment
	o   Seniority & Strategic Level
	o   Impact & Results
	o   Relevance to role
	o   Recency
	o   Uniqueness / Differentiation

-          A Decision column indicating:
	o   Must stay
	o   Merge IDx and IDy
	o   Can be removed
-          A comment column indicating the decision column

Based on this table we will be able to have an overview off how we can in a balanced way keep the most relevant bullets While still being thorough in covering core and important skill connected bullets.
